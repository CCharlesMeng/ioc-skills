## 1. 规范要求



**详情链接**: [WIKI2023041000672 - 基础规范](https://clouddevops.huawei.com/domains/29585/wiki/2104/WIKI2023041000672)



### 1.1 SQL编码规范



| 规范项 | 说明 |

|--------|------|

| **表别名** | 必须有别名，且具备可读性，禁止使用`t1`、`t2`等无意义别名 |

| **字段引用** | 引用表字段时需加表别名，避免加字段后歧义导致任务失败 |

| **注释描述** | 子查询和复杂逻辑计算需有注释描述 |

| **缩进换行** | CASE-WHEN需缩进换行，四空格为一个缩进量 |

| **NULL值处理** | 判断维度字段空值需考虑NULL与空字符串，度量字段为NULL必须处理为0 |

| **禁用select \*** | 查询时指定查询列，避免因加字段导致任务失败 |

| **禁用嵌套超3层** | 嵌套查询超过3层会导致任务变慢且不利于维护 |



### 1.2 性能优化规范



| 规范项 | 说明 |

|--------|------|

| **行裁剪** | 表关联前使用where条件过滤数据，避免全表扫描 |

| **分区字段过滤** | 对于分区表，尽量使用分区字段过滤 |

| **减少Job数量** | Join key相同会合并为一个MapReduce任务（最多16张表join） |

| **数据倾斜** | 对关联字段空值做特殊转换，避免数据倾斜 |

| **笛卡尔积** | 避免使用笛卡尔积，必须使用时用`cross join`写法 |



### 1.3 数据表开发规范



| 规范项 | 说明 |

|--------|------|

| **表名前缀** | 根据数据分层增加前缀 `ods_/dwd_/dws_/ads_` |

| **字段名** | 使用下划线【_】作为单词连接 |

| **存储格式** | Hive表建表存储格式统一为ORC格式 |

| **试算表** | Hive试算表统一放到`bi_test`库 |

| **DDL脚本命名** | 格式：`DDL-[database]-[tablename]` |



### 1.4 计算任务命名规范



| 规范项 | 格式 |

|--------|------|

| **计算任务** | `CT-COMMON-[任务类型]-[名称]` |

| **工作流** | `WF-COMMON-[任务类型]-[名称]` |



**任务类型枚举**:

- `FORMAL` - 正式任务

- `TEST` - 测试任务  

- `DATALAB` - 数据实验室

- `TRIAL` - 试算

- `RECOVER` - 恢复

## 3. 脚本开发



**详情链接**: [WIKI2023031400228 - 脚本开发](https://clouddevops.huawei.com/domains/29585/wiki/2104/WIKI2023031400228)



### 3.1 新建脚本流程



| 步骤 | 操作 |

|------|------|

| 1 | 在本地代码仓`dp_script`目录下新建SQL脚本 |

| 2 | 进入CBCDEV空间，选择脚本类型新建脚本 |

| 3 | 将开发好的代码复制并保存提交 |



### 3.2 脚本存放目录



| 类型 | 中国站路径 | 国际站路径 |

|------|------------|------------|

| **脚本** | `code/China/release/DataCode/dp_script` | `code/Internation/release/DataCode/dp_script/01_DataAnalyse` |

| **作业** | `code/China/release/DataCode/dp_job` | `code/Internation/release/DataCode/dp_job` |

| **建表语句** | `code/China/release/DataCode/table` | `code/Internation/release/DataCode/dp_script/02_Table` |



### 3.3 建表DDL开发



**Hive建表示例**:

```sql

CREATE TABLE `bi_temp.example_table_test_20230410` (

    `col1` string COMMENT '字段一',

    `col2` string COMMENT '字段二'

) COMMENT '示例临时表'

PARTITIONED BY (`col9` string COMMENT '字段九')

ROW FORMAT SERDE 'org.apache.hadoop.hive.ql.io.orc.OrcSerde'

STORED AS ORC;

```



**注意事项**:

- 存储格式选择ORC格式

- 分区表加字段需加CASCADE关键字

- 建表脚本存放于`dp_script/02_Table`目录



### 3.4 特殊场景



| 场景 | 处理方式 |

|------|----------|

| DLI脚本 | 升级成功后手动修改队列 |

| 同步现网脚本 | 从现网导出压缩包，上传到CBCDEV/CBCTEST |

| 同步环境变量 | 手动在三个环境分别创建 |

## 4. 作业开发



**详情链接**: [WIKI2023031400230 - 作业开发](https://clouddevops.huawei.com/domains/29585/wiki/2104/WIKI2023031400230)



### 4.1 新建作业流程



| 步骤 | 操作 |

|------|------|

| 1 | 进入CBCDEV空间 |

| 2 | 选择目录新建作业（一般批处理作业） |

| 3 | 拖拽节点库算子创建作业节点 |

| 4 | 设置调度方式和参数配置 |

| 5 | 保存提交作业 |



### 4.2 作业规范



| 规范项 | 说明 |

|--------|------|

| **优先级** | 影响重点资产设为高，不影响禁止设为高 |

| **超时时长** | 根据历史平均执行时长+20min设置 |

| **调度时间** | 黄金时段00:00~08:00需评审 |

| **业务标签** | 根据业务属性添加，与存放目录一致 |



### 4.3 支持流水线升级的算子



- HiveSQL算子

- DWS SQL算子

- CDM算子

- DLI算子

- 一键入湖算子



**不支持算子**: 其他算子需手工升级

**Commit Message模板**:

**注意事项**:

1. 不能有空格

2. 单号需要为有效单号

3. 不能将测试代码合入Dev分支

| **RELEASE_VERSION** | 每次不同，递增加1 |

| **分支** | 选择dev分支 |

## 6. 试算验证（A/B表）



**详情链接**: [WIKI2023042701583 - 试算验证](https://clouddevops.huawei.com/domains/29585/wiki/2104/WIKI2023042701583)



### 6.1 A/B表概念



| 表类型 | 说明 |

|--------|------|

| **A表** | 现网正式表 `dwd_cdr.t_xxx` |

| **B表** | 现网试算表 `bi_test.t_xxx_gray` |



### 6.2 试算流程



| 步骤 | 操作 |

|------|------|

| 1 | 在现网创建临时试算表（库名`bi_test`） |

| 2 | 修改代码中表名为试算表名 |

| 3 | 添加参数`queuename`和`job.name` |

| 4 | 创建试算任务（前缀`CT-COMMON-TEST`） |

| 5 | 拉起试算任务运行数据 |



### 6.3 数据核对要点



- B表数据与预期逻辑一致

- A表与B表数据量一致

- A表与B表关键指标一致
