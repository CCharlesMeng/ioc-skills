# 需求规格: 工单管理

> 由 ioc-spec 生成 | 2026-04-14

## 需求背景

运维团队需要一个统一的工单系统来管理内部 IT 服务请求，替代当前通过邮件和即时通讯分散处理的方式。

## 用户角色

| 角色 | 说明 |
|---|---|
| 提交人 | 提交工单并跟踪处理进度的普通员工 |
| 处理人 | 被分配工单并负责解决的运维人员 |
| 管理员 | 分配工单、查看全局状态的运维主管 |

## 功能范围

### 做

- 工单的创建、查看、筛选
- 工单状态流转（待分配 → 处理中 → 已完成/已关闭）
- 按状态、优先级筛选和关键词搜索
- 工单详情查看

### 不做

- 工单处理人自动匹配/智能派单
- 工单 SLA 时效管理
- 工单统计报表和数据分析
- 工单模板管理

## 用户场景

**场景 A: 员工提交工单**
员工进入工单管理页面，点击"新建工单"，在弹出的表单中填写标题、选择优先级和类型、描述需求详情，可选上传附件，提交后工单进入"待分配"状态。

**场景 B: 管理员查看和筛选工单**
管理员进入工单列表页，通过状态和优先级下拉筛选器缩小范围，或输入工单编号/标题关键词搜索，快速定位目标工单并点击查看详情。
