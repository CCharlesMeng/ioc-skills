# Sketch-Decode 4 节 Schema

每张截图必须按以下 4 节逐节填写。不允许跳过任何节——如果某节确实无内容（如模态弹窗没有导航），写"无"并说明原因。

> **DSL 设计原则**：缩进嵌套 + 关键词标注，与 [Wireweave](https://wireweave.org/) 思路一致，但用缩进代替花括号以降低弱模型的生成难度。关键词表可按项目扩展。

## 1. 布局还原

**指令**：用布局 DSL 还原截图的页面结构和所有可见内容。

**要求**：
- 用缩进表示嵌套关系（父子层级）
- 用元素关键词标注类型（见下方关键词表）
- 所有可见文案原文直接写在元素后面
- 样式和状态用 `()` 标注在文案后
- 如果文字模糊不清，就地标注 `(模糊)`
- 不需要对齐字符，只需要缩进正确

### 元素关键词表

**结构容器**：

| 关键词 | 含义 | 用法 |
|---|---|---|
| `page` | 整页 | `page "页面标题"` |
| `modal` | 模态弹窗 | `modal "弹窗标题"` |
| `nav` | 导航栏 | `nav` 后列出菜单项 |
| `header` | 页头区域 | `header (row)` |
| `footer` | 页脚区域 | `footer (row)` |
| `sidebar` | 侧边栏 | `sidebar (left)` / `sidebar (right)` |
| `grid` | 网格布局 | `grid 3x2` = 3列2行；子元素用 `cell` 包裹 |
| `cell` | 网格单元格 | `cell (span=2)` 表示跨 2 列 |
| `card` | 卡片容器 | `card "卡片标题"` |
| `tabs` | 标签页组 | `tabs` 后列出 `- 标签名 (active)` |
| `form` | 表单 | 包裹表单元素 |
| `filters` | 筛选区 | `filters (row)` |
| `section` | 通用分区 | 无特定语义时的容器 |

**基础元素**：

| 关键词 | 含义 | 用法 |
|---|---|---|
| `text` | 静态文字 | `text "页面标题"` |
| `btn` | 按钮 | `btn "文案" (primary)` / `(ghost)` / `(danger)` |
| `link` | 文字链接 | `link "查看"` |
| `input` | 输入框 | `input "占位符文字"` |
| `textarea` | 多行输入 | `textarea "占位符" (4行)` |
| `select` | 下拉框 | `select "标签" = "当前值"` |
| `upload` | 文件上传 | `upload "提示文字"` |
| `img` | 图片/图标 | `img "描述"` |
| `table` | 表格 | 下挂 `head:` 和 `row:` |
| `---` | 水平分隔 | 区域之间的视觉分隔 |

**数据可视化**：

| 关键词 | 含义 | 用法 |
|---|---|---|
| `kpi` | 数值指标卡 | `kpi "指标名" = "数值" (trend=+12%)` |
| `chart.bar` | 柱状图 | `chart.bar "图表标题"` 后列出可见数据项 |
| `chart.line` | 折线图 | `chart.line "趋势图标题"` |
| `chart.pie` | 饼图/环形图 | `chart.pie "占比图标题"` |
| `chart.gauge` | 仪表盘 | `chart.gauge "指标名" = "78%"` |
| `chart.map` | 地图 | `chart.map "地图标题"` |
| `chart.scatter` | 散点图 | `chart.scatter "散点图标题"` |
| `chart.radar` | 雷达图 | `chart.radar "雷达图标题"` |
| `chart.heatmap` | 热力图 | `chart.heatmap "热力图标题"` |
| `rank` | 排行榜 | `rank "排行标题"` 后列出可见条目 |
| `progress` | 进度条 | `progress "标签" = "75%"` |
| `ticker` | 滚动数字/文字 | `ticker "实时数据"` |
| `legend` | 图例 | `legend` 后列出图例项 |

### 布局修饰词

| 修饰词 | 含义 |
|---|---|
| `(row)` | 子元素水平排列 |
| `(right)` / `(left)` / `(center)` | 对齐方式 |
| `(active)` | 当前选中/高亮 |
| `(必填)` | 必填标记 |
| `(dark)` | 深色/暗色背景 |
| `(fullscreen)` | 全屏模式 |
| `(span=N)` | 网格跨列数 |
| `(red)` `(blue)` `(gray)` `(green)` 等 | 颜色标注 |
| `(tag)` | 标签样式 |
| `(trend=+N%)` / `(trend=-N%)` | 趋势标注（用于 KPI） |

### 图表数据记录规则

图表内部只记录**可见的**数据标签和数值，不需要列出完整数据集：

## 2. 组件识别

**指令**：将布局中的结构区域映射到项目组件文档（如有），描述其功能特征。

**要求**：
- 每个区域标注：功能描述 + 匹配组件（如果项目提供了组件文档）+ 匹配度
- 功能描述使用用户语言，不使用前端技术术语
- 如果项目没有组件文档，只写功能描述，不做匹配
- 匹配度分三档：完全匹配 / 需扩展 / 无现有组件

**有组件文档时的示例**：

| 页面区域 | 功能描述 | 匹配组件 | 匹配度 |
|---|---|---|---|
| kpi 区域 | 6 个数值指标卡，含趋势箭头和颜色状态 | KPI指标卡 | 完全匹配 |
| chart.line | 近7日工单趋势，双折线（新增/完成） | 趋势折线图 | 完全匹配 |
| chart.pie | 4 类工单占比饼图 | 占比图 | 完全匹配 |
| chart.map | 全国节点分布，4 个节点含状态颜色 | 地图标注 | 需扩展（增加状态色） |
| rank | TOP5 处理排行，序号 + 姓名 + 完成数 | 排行榜 | 完全匹配 |
| table (实时告警) | 4 列告警表，含级别标签和状态 | 数据表格 | 完全匹配 |

**无组件文档时的示例**：

| 页面区域 | 功能描述 |
|---|---|
| kpi 区域 | 6 个数值指标卡，含趋势箭头和颜色状态 |
| chart.line | 近7日工单趋势，双折线（新增/完成） |
| chart.map | 全国节点分布图，4 个节点含红绿黄状态色 |

**组件文档要求**：

项目如果提供组件文档供本 skill 匹配，文档应遵循以下约定：
- 用功能语言描述组件，不暴露技术实现
- 每个组件包含：名称（功能化命名）、功能说明、适用场景、支持的变体
- 文档路径由项目自定义，在 skill 调用时通过用户告知或 MCP 读取

## 3. 交互与状态

**指令**：列出截图中所有可交互的元素及其关联的状态标识。

**要求**：
- 将交互元素与其关联的状态/反馈归为一组
- 每个元素标注：位置 + 视觉样式 + 可见的状态值（如有）
- 不描述点击后的预期行为（那是 ioc-spec 的工作），只记录当前可见状态
- 如果某个状态标识不关联任何交互元素，单独列出
- 数据大屏中图表的悬浮提示框（tooltip）如果可见则记录，不可见不猜测

**示例（数据大屏）**：

```
交互元素：
- chart.line "工单趋势" 可见 tooltip 显示"周三: 新增15, 完成18"

- rank 各条目可能可点击（无明显链接样式，不确定）

状态标识：
- kpi "待处理" 数值红色，表示异常

- kpi "今日工单" 趋势箭头 ↑ +12%

- table 告警级别：严重(red,tag) / 警告(orange,tag) / 信息(blue,tag)
```

## 4. 解析自检

**指令**：检查本次解析的完整性和确定性，主动暴露问题。

**要求**：
- 列出所有不确定或模糊的区域
- 列出因截图分辨率/裁剪导致的信息缺失
- 列出收起状态的下拉框、折叠面板等无法看到完整内容的元素
- 数据大屏注意：图表是否有被遮挡的数据标签、动态效果是否静态截图无法体现
- 如果所有内容都清晰完整，写"所有区域解析完整，无模糊或遗漏项"

**示例（数据大屏）**：

```
- chart.line 仅显示近7日数据，是否支持切换时间范围不可见
- chart.map 仅标注 4 个城市节点，是否有更多节点被地图缩放层级隐藏未知
- chart.map 节点状态颜色推测 green=正常/yellow=告警/red=故障，无图例确认
- rank 仅显示 TOP5，完整排行长度不可见
- table "实时告警" 显示 3 行，注释"共12条"，是否有滚动/分页不确定
- 页面右上角时间是否为实时刷新，刷新频率无法从静态截图判断
- 整体深色背景下部分文字对比度较低，"平均响应"数值可能为 4.2 或 4.7(模糊)
```
