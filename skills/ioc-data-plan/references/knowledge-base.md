## 一、ETL 模式库

常见的数据转换模式，供 Phase 2 澄清时推荐使用。

### P-001: 日度快照聚合

- **场景**：从明细表生成日度汇总指标（如当日工单总数、当日完成数）
- **聚合方式**：按日期分组，对目标字段做 COUNT / SUM / AVG
- **注意事项**：
  - 时区处理：确认"一天"的起止时间（通常为 00:00:00 ~ 23:59:59 本地时间）
  - 跨天记录归属：创建时间 vs 完成时间，取哪个时间字段决定归属
  - 数据延迟：源表数据到达可能有延迟，需确认 T+0 还是 T+1

### P-002: 滑动窗口聚合

- **场景**：计算"近N天"类指标（如近7日趋势、近30日均值）
- **聚合方式**：按日期分组后，窗口函数或子查询取近N天范围
- **注意事项**：
  - "近7日"是否含今天——通常不含（因为今天数据不完整），需澄清
  - 节假日/非工作日是否跳过——通常不跳过（自然日），需澄清
  - 窗口起止：`[today-7, today-1]` vs `[today-6, today]`

### P-003: 比率/占比计算

- **场景**：计算百分比类指标（如 SLA 达标率、完成率、占比）
- **聚合方式**：分子指标 / 分母指标 × 100
- **注意事项**：
  - 分母为零时的处理：返回 NULL / 0 / 特殊标记
  - 精度：保留几位小数
  - 口径陷阱：分子分母的筛选条件必须一致（同一时间范围、同一状态筛选）

### P-004: 同比/环比计算

- **场景**：计算同比增长率、环比变化率
- **聚合方式**：(本期 - 上期) / 上期 × 100
- **注意事项**：
  - 上期为零时的处理
  - 同比 vs 环比的周期定义（日同比=昨日、周同比=上周同天、月同比=上月同天）
  - 首次出现（无历史数据）时的展示策略

### P-005: 维度下钻聚合

- **场景**：指标需要按多个维度（区域、类型、优先级等）拆分展示
- **聚合方式**：GROUP BY 维度字段
- **注意事项**：
  - 维度值的标准化（如区域名称是否统一）
  - "其他"兜底维度的处理
  - 维度组合爆炸：多维度交叉时行数膨胀

## 二、领域指标模板

特定业务领域的常见指标体系，供 Phase 1 指标盘点和 Phase 2 澄清时参考。

### C-001: IOC 运维大屏 — 工单类

- **常见指标**：工单总数、新增工单数、待处理数、已完成数、SLA 达标率、平均响应时间、平均处理时长
- **常见维度**：时间（日/周/月）、区域、工单类型、优先级、处理人
- **口径注意**：
  - "待处理"的定义因项目而异：未分配 vs 已分配未开始 vs 处理中
  - SLA 的"达标"标准因合同而异：响应时间 < X小时 and/or 解决时间 < Y小时
  - "平均响应时间"的起止点：创建时间 → 首次响应时间

### C-002: IOC 运维大屏 — 人力资源类

- **常见指标**：总人力、配置率、本地化率、人均效能、人力结构分布
- **常见维度**：组织层级、区域/国家、人员类型（自有/外包）
- **口径注意**：
  - "总人力"是在编人数还是在岗人数
  - "配置率"= 实际人数 / 编制人数，编制数据来源需确认
  - 人员分类标准因项目而异

### C-003: IOC 运维大屏 — 设备/资源类

- **常见指标**：设备总数、在线率、告警数、故障率、资源利用率
- **常见维度**：设备类型、区域、机房、告警级别
- **口径注意**：
  - "在线率"的计算周期：瞬时值 vs 时间段内的平均在线率
  - "故障率"= 故障设备数 / 设备总数 vs 故障次数 / 设备总数
  - 告警去重：同一设备同一故障的重复告警是否合并
