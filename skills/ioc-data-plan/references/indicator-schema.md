# 存量指标解析与匹配规范

本文档是 Phase 0 解析 Excel 和 Phase 1 交叉比对的**单一真源**。

---

## 一、Excel 解析

### 预期结构

每行一个指标。实际列名可能因项目而异——Phase 0 读取时需做列名映射。

#### 核心列（必须存在）

| 列 | 说明 | 示例 |
|---|---|---|
| 指标名称 | 指标的业务名称 | 今日工单总数 |
| 应用表 | 指标所属的应用表名 | app_workorder |
| 字段名 | 指标在应用表中的字段名 | total_count |

#### 扩展列（有则解析，无则跳过）

| 列 | 说明 | 示例 |
|---|---|---|
| 口径说明 | 指标的计算口径描述 | COUNT(status != '已关闭') |
| 数据类型 | 字段的数据类型 | INT / DECIMAL / VARCHAR |
| 时间粒度 | 指标的聚合周期 | 日 / 周 / 月 / 实时 |
| 维度 | 指标的切面维度 | 区域 / 类型 / 无 |
| 源表 | 数据来源的源表名 | ods_workorder |
| 主题表 | 中间层的主题表名 | dws_workorder_daily |
| 更新频率 | 数据更新的频率 | 每日凌晨 / 实时 |
| 备注 | 其他补充信息 | 含历史工单 |

### 列名映射

Phase 0 读取时按以下策略映射：

1. **精确匹配**：列名与上述完全一致 → 直接映射
2. **模糊匹配**：列名包含关键词 → 尝试映射，列出映射结果请用户确认
3. **无法匹配**：列名无法识别 → 忽略该列，记录在输入装载日志中

模糊匹配关键词表：

| 目标列 | 可能的列名变体 |
|--------|--------------|
| 指标名称 | 指标名、名称、指标、indicator_name、metric_name |
| 应用表 | 表名、app_table、table_name、归属表 |
| 字段名 | 字段、field、column、column_name |
| 口径说明 | 口径、计算逻辑、算法、calculation、formula |
| 数据类型 | 类型、type、data_type、字段类型 |
| 时间粒度 | 粒度、granularity、周期、period |
| 源表 | 来源表、source_table、ods表、数据来源 |

### 失败路径

| 情况 | 处理方式 |
|------|---------|
| 核心列缺失（缺指标名称 / 应用表 / 字段名） | 阻断解析，列出缺失列，请用户补充或手动提供指标清单 |
| Excel 为空或无有效数据行 | 等同无 Excel，进入降级 A 模式 |
| 合并单元格 | 尝试展开合并区域取左上角值；展开失败时跳过该行，记录警告 |
| 多 Sheet | 每个 Sheet 独立解析，Sheet 名作为分组标签 |
| 编码异常 / 加密工作簿 | 阻断解析，提示用户导出为 CSV 或解密后重新提供 |
| CSV 替代 | 接受 CSV 输入，按相同规则解析（分隔符自动检测） |

### 人工输入回退

如果用户无法提供 Excel/CSV，接受以下替代格式：

```
请按以下格式提供存量指标（每行一个）：

指标名称 | 应用表 | 字段名 | 口径说明（可选）
今日工单总数 | app_workorder | total_count | COUNT(status != '已关闭')
```

### 指标索引构建

解析完成后，构建内存中的指标索引，供 Phase 1 交叉比对使用：

```
指标索引结构：
{
  "<应用表>.<字段名>": {
    name: "指标名称",
    table: "应用表",
    field: "字段名",
    definition: "口径说明（如有）",
    type: "数据类型（如有）",
    granularity: "时间粒度（如有）",
    dimensions: "维度（如有）",
    source_table: "源表（如有）",
    theme_table: "主题表（如有）"
  }
}
```

索引键为 `<应用表>.<字段名>`，保证唯一性。

#### 多 Sheet 冲突处理

不同 Sheet 中相同 `<应用表>.<字段名>` 视为同一指标（取第一次出现的），在解析日志中记录冲突。

## 二、匹配打分规则

Phase 1 将数据需求项与指标索引比对时，按以下规则逐项打分。

### 评分维度

| 维度 | 条件 | 得分 |
|------|------|------|
| 名称精确匹配 | 数据需求项名称与指标名称完全一致（去除空格后比较） | +5 |
| 名称包含匹配 | 指标名称包含数据需求项的核心词，或反之 | +3 |
| 应用表一致 | 数据需求项的上下文（所属页面/区域）与指标的应用表归属一致 | +3 |
| 口径语义一致 | 指标的口径说明与数据需求项的语义描述一致 | +2 |
| 时间粒度一致 | 指标的聚合周期与需求一致 | +1 |
| 维度切面一致 | 指标的维度与需求一致 | +1 |

最高可能得分：15。

### 分类阈值

| 总分 | 分类 | 含义 |
|------|------|------|
| ≥ 8 | 复用 | 高置信度匹配，Phase 3 直接引用 |
| 4 ~ 7 | 修改 | 有近似指标但口径/维度/粒度有差异，Phase 2 澄清差异 |
| 0 ~ 3 | 新增 | 无有效匹配，Phase 2 重点澄清 |

### 并列候选处理

如果同一数据需求项有多个候选指标得分相同（≥ 4）：

1. 在指标清单的"差异说明"列列出所有候选及其得分
2. 标注 `[待确认:多候选]`，由用户在 Phase 1 确认门决定取哪个

### 匹配结果呈现

指标清单表必须包含"匹配分"列，使分类依据可审查：

```markdown
| # | 页面组件 | 数据需求 | 匹配指标(Excel) | 分类 | 匹配分 | 差异说明 |
|---|---------|---------|----------------|------|--------|---------|
| D-1 | kpi "今日工单" | 当日工单总数 | app_workorder.total_count | 复用 | 10 | — |
| D-2 | chart.line "趋势" | 近7日新增/完成 | — | 新增 | 0 | 无存量指标 |
| D-3 | kpi "SLA达标率" | SLA达标百分比 | app_sla.rate | 修改 | 6 | 现有口径为月度，需改日度 |
```

### 无 Excel 时的处理

降级 A 模式下（无存量指标），所有数据需求项直接标记为"新增"，匹配分为 0，跳过匹配步骤。
