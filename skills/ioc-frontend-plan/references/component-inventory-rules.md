# 存量组件盘点规则

这是 `ioc-frontend-plan` 中组件扫描、证据判定、分类和审批的唯一真源。

---

## 扫描范围

### 优先扫描

按以下顺序查找 Angular 工作区中的共享组件来源：

1. `angular.json` / `workspace.json`
2. 各项目 `project.json`
3. 根目录 `tsconfig*.json` 中的 path aliases
4. 工作区级目录：
   - `libs/`
   - `projects/`
   - `src/app/shared/`
   - `src/app/common/`
   - `src/shared/`

### 提取信息

每个候选组件至少提取：

| 字段 | 用途 |
| --- | --- |
| 类名 / selector | 建立候选索引 |
| Inputs / outputs | 判断接口覆盖度 |
| Provider / 依赖 | 判断是否带业务耦合 |
| 模板摘要 | 判断结构相似度 |
| 使用位置 | 判断是否真为共享组件 |
| 示例/文档/Storybook | 增强证据可信度 |

## 证据等级

组件复用不是“看起来像”就够，必须有证据。

| 等级 | 证据 | 说明 |
| --- | --- | --- |
| A | 已有同类业务场景的真实调用 | 最强证据 |
| B | Storybook、示例页、明确文档 | 强证据 |
| C | selector、inputs/outputs、模板摘要可匹配 | 中等证据 |
| D | 仅名称或注释相似 | 弱证据，不足以直接判复用 |

规则：

- `复用` 至少需要 `A` 或 `B`，或多个 `C` 共同支持
- 只有 `D` 时，不能直接判 `复用`

## 三分类

### 复用

判定条件：

- 功能语义匹配
- 接口覆盖需求
- 不需要改组件代码
- 证据等级满足复用门槛

输出：

- 分类为 `复用`
- 不需要开发者审批

### 扩展

判定条件：

- 候选组件方向正确
- 但需要新增 input / output / 模板 / 内部逻辑
- 且改动是“增量扩展”，不是重写

输出：

- 分类为 `扩展`
- 必须记录改动点、兼容性影响、回归范围
- 必须经过开发者明确同意

### 新增

判定条件：

- 没有合适候选
- 或要达成需求必须对存量组件做破坏性改动
- 或扩展成本已经高于直接新建

输出：

- 分类为 `新增`
- 必须明确归属：共享组件 / 页面内组件
- 必须经过开发者明确同意

## 审批矩阵

| 情况 | 需要审批 | 阻断文档生成 | 阻断直接编码 |
| --- | --- | --- | --- |
| 复用 | 否 | 否 | 否 |
| 扩展共享组件 | 是 | 否 | 是 |
| 新增共享组件 | 是 | 否 | 是 |
| 新增页面内组件 | 是 | 否 | 是 |

解释：

- **可以先出文档**：方案文档允许带 `[待确认:需开发者审批]`
- **不能直接编码**：只要扩展/新增项未审批，就不能把方案表述为“可直接开工”

## 审批输出格式

Phase 1 应输出统一审批表，而不是散落在正文中：

```markdown
| C-ID | 分类 | 候选/目标组件 | 路径 | 改动或归属 | 审批状态 | 是否阻断编码 |
| --- | --- | --- | --- | --- | --- | --- |
| C-2 | 扩展 | LineChartComponent | libs/shared-ui/src/lib/line-chart/ | 增加多序列输入 | 待确认 | 是 |
| C-3 | 新增 | NodeMapComponent | -（新建） | 页面内组件 | 已确认 | 否 |
```

可接受的审批状态：

- `已确认`
- `待确认`
- `拒绝`
- `同意但调整方案`

## 盘点流程

1. 从 `Sketch-Decode.md` 和 `Spec.md` 提取组件需求。
2. 建立候选组件清单。
3. 给每个候选打证据等级。
4. 判定 `复用 / 扩展 / 新增`。
5. 把所有 `扩展 / 新增` 输出到审批矩阵。
6. 未审批项进入文档时统一标记 `[待确认:需开发者审批]`。

## 降级模式

### 无法访问项目代码

- 不做自动扫描
- 允许用户手动提供组件清单
- 若无清单，则所有项默认视为 `新增`

### 无法判断证据等级

- 默认下调一档
- 不允许因为信息不足直接判 `复用`

### monorepo 结构复杂

- 优先相信 workspace 配置和 path alias
- 不只扫描 app 的 `sourceRoot`

## 与 Sketch-Decode 组件识别的关系

`Sketch-Decode.md` 里的“组件识别”是弱匹配参考，不是最终结论。

| 来源 | 作用 |
| --- | --- |
| Sketch-Decode 组件识别 | 提供初步候选 |
| 本文件规则 | 给出最终分类与审批要求 |
